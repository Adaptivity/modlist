Modlist API

API Requests

API Requests use api.php on the root of the site (http://modlist.mcf.li/api.php).
Requests are executed by attaching a GET request in the following form:

Key Value Type Accepted Values
request string hash, json
version string 1.5.1, 1.5.0, 1.5, 1.4.7, 1.4.6, 1.4.7_1.4.6, 1.4.6_1.4.7, 1.4.5, 1.4.4, 1.4.5_1.4.4, 1.4.4_1.4.5, 1.4.2

Example Usage

http://modlist.mcf.li/api.php?version=1.5.1&request=hash

Hash Request

Returns an md5 check of the requested JSON file.

Example Result

fbf9cc4a8704a28045037a69908b2b97

JSON Request

Format

[
  {
    "name":"",
    "other":"",
    "link":"",
    "desc":"",
    "author":"",
    "type":[
      ""
    ],
    "dependencies":[
      ""
    ],
    "versions":[
      ""
    ]
  }
]

Fields name, other, link, desc and author can contain any string.
The type array can hold one or more of the following values:

  • Universal
  • SSP
  • SMP
  • LAN
  • Client
  • Server

The dependencies array can either be empty (treated as No Dependency) or hold one or more of the following values:
  • Modloader
  • Not Forge Compatible
  • Forge Compatible
  • Forge Required

It may also contain other dependencies such as iChun Utils.
